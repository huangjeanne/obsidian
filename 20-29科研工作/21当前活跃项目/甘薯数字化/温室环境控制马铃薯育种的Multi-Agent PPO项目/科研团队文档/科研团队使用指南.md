## 🎓 团队成员介绍

### 核心团队（11名专家）

| 角色 | 名称 | 职责 |
|------|------|------|
| 🎯 首席研究员 | principal-investigator | 统筹研究方向、协调各角色工作、把控研究质量与进度 |
| 📚 文献挖掘专家 | literature-miner | 从arXiv/PubMed/bioRxiv等数据库挖掘农业智能体、强化学习、多智能体系统等相关论文 |
| 🔍 研究空白分析师 | gap-analyst | 识别现有研究的局限性和潜在研究方向 |
| 🧩 知识整合专家 | knowledge-synthesizer | 整合多篇论文观点形成系统性知识图谱与综述框架 |
| 🧪 实验设计师 | experimental-designer | 设计研究实验方案、数据采集策略、验证方法 |
| 🏗️ 算法架构师 | algorithm-architect | 设计AI模型架构、智能体决策框架、推理流程 |
| ⚙️ 训练指导专家 | training-advisor | 提供模型训练建议、超参数调优策略、性能优化方案 |
| 📊 数据分析专家 | data-analyst | 分析实验结果、统计检验、解读数据含义 |
| 🎨 可视化设计师 | visualization-designer | 创建数据可视化图表、论文配图、技术示意图 |
| ✍️ 学术写作专家 | academic-writer | 撰写论文各章节、润色语言、确保符合学术规范 |
| 📖 文献管理专家 | citation-manager | 管理参考文献、确保引用格式正确、构建参考文献列表 |

## 💡 最佳实践

### 1. 保持研究上下文

在 `研究上下文.md` 文件中记录：
- 当前研究进度
- 已完成的工作
- 下一步计划
- 遇到的问题

### 2. 组织项目文件

建议的文件结构：

```
项目根目录/
├── 科研团队文档/
│   ├── 科研团队使用指南.docx
│   └── 研究上下文.docx
├── 文献库/
│   ├── 已读论文/
│   └── 待读论文/
├── 实验数据/
│   ├── 原始数据/
│   ├── 处理后数据/
│   └── 结果分析/
├── 草稿/
│   ├── 论文草稿/
│   ├── 实验方案/
│   └── 会议记录/
└── 代码/
    ├── 模型/
    ├── 训练脚本/
    └── 分析脚本/
```

### 3. 重新激活时的最佳沟通方式

```
重新激活 agri-research-team，继续之前的研究。

项目路径：[你的项目路径]
当前进度：[简述已完成的工作]
下一步计划：[说明需要团队做什么]
特殊要求：[任何特定需求或约束]
```

**示例：**

```

下一步计划：分析实验结果，优化算法，准备撰写论文。
特殊要求：重点关注马铃薯育种场景的特异性。
```

## 📋 工作流程示例

### 完整的研究流程

```
1. PI 启动研究 → 明确研究问题和目标
2. 文献挖掘 → 收集相关论文
3. 研究空白分析 → 找到创新点
4. 知识整合 → 形成理论框架
5. 实验设计 → 设计验证方案
6. 算法设计 → 构建模型架构
7. 训练指导 → 优化模型性能
8. 数据分析 → 解读实验结果
9. 可视化 → 创建论文配图
10. 学术写作 → 撰写论文
11. 文献管理 → 整理参考文献
12. PI 审核 → 最终把关
```

### 部分调用示例

**只需要文献调研：**
```
文献挖掘专家，帮我收集最近一年关于多智能体强化学习在温室控制中应用的论文
```

**只需要算法设计：**
```
算法架构师，基于我的实验数据，设计一个适合马铃薯育种场景的PPO变体
```

**只需要数据分析：**
```
数据分析专家，帮我分析训练日志，找出性能瓶颈
```
