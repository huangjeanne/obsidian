## 一、研究背景与问题陈述

### 核心问题（通俗解读）

这项研究要解决的根本问题是：**马铃薯科研领域存在"信息混乱"和"查阅困难"的难题，严重阻碍了科学家培育更好土豆品种的进程。**

具体来说，科学家面临三大困境：

#### 困境1：基因的"身份证"太乱了（命名混乱）

- 马铃薯的参考基因组版本不断更新（从早期版本到DMv8.1）
- 每次更新，基因的命名规则就变一次
- 以前的论文里说的基因，现在找不到对应位置
- **典型案例**：`SP5G-B`基因曾被错误地标记为`SP6A`
- **类比**：这就像你朋友的电话号码每年都在变，而且还会被标到别人的名字下

#### 困境2：论文太多，根本读不过来（信息过载）

- 过去10年，马铃薯论文数量**翻了一倍**
- 在PubMed搜索"potato"会得到**32,000多篇论文**
- 其中约**83.5%**只是顺带提到"土豆"，并非专门研究
- 科学家需要花**数周时间**筛选和阅读文献

#### 困境3：普通AI不懂"土豆行话"（AI幻觉）

- ChatGPT等通用AI没有专门学过马铃薯知识
- 问专业问题容易"一本正经地胡说八道"（专业术语叫"幻觉"）
- 给出的答案往往不准确，也没有引用来源
- 科学家**不敢完全信任**这些通用AI

### 数据处理流程

#### 第一阶段：高质量文献数据获取与预处理

**目标**：从海量文献中清洗出纯净的马铃薯科研数据

| 步骤 | 操作 | 工具/方法 | 结果 |
|------|------|----------|------|
| 1. 数据源获取 | PubMed检索 (1900-2025) | 关键词"Potato" | 32,000+篇论文 |
| 2. 智能筛选 | 阅读标题和摘要 | DeepSeek-R1 (671B) | 剔除红薯等非相关文献 |
| 3. 人工补录 | 补救无摘要文章 | 人工审查 | 补录105篇重要文献 |
| 4. 质量控制 | 期刊分区筛选 | CAS Q1/Q2标准 | 3,233篇核心文献 |
| 5. 格式转换 | PDF转纯文本 | MinerU (v1.2.2) | 机器可读文本 |

#### 第二阶段：功能基因数据库构建

**目标**：解决"基因命名混乱"问题，建立统一基因"户口本"

| 步骤 | 操作 | 工具/方法 | 结果 |
|------|------|----------|------|
| 1. 信息提取 | 提取基因ID和符号 | DeepSeek-V3 + Qwen3-235B | 4,221对ID-符号 |
| 2. 人工校验 | 核对准确性 | 人工审查963篇论文 | 确保数据准确 |
| 3. 基因映射 | 统一到DMv8.1基因组 | MCscan + BLAST+ | 2,571个非冗余基因 |
| 4. 数据存储 | 存入数据库 | SQLite | 快速查询系统 |

**核心难点：基因ID映射**

- **方法一**：共线性分析
  - 工具：JCVI toolkit (v1.3.6) 中的 MCscan
  - 适用：DMv4.03, DMv6.1, SolTub3.0, RH89-039-16等

- **方法二**：序列比对
  - 工具：NCBI BLAST+ (v2.14.0)
  - 阈值：相似度 >90%
  - 适用：Genbank, Uniprot, SGN, Agria等

#### 第三阶段：文献知识库与RAG架构

**目标**：实现"精准问答"，避免AI幻觉

```
RAG（检索增强生成）工作流程：
┌─────────────────────────────────────────────────┐
│ 1. 向量化 (Vectorization)                        │
│    • 文献切分为段落                              │
│    • SnowFlake模型转为向量                       │
│    • 存储在向量数据库中                          │
└─────────────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────────────┐
│ 2. 检索 (Retrieval)                              │
│    • 用户问题转为向量                            │
│    • 计算余弦相似度                              │
│    • 初步筛选前100个相关段落                     │
└─────────────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────────────┐
│ 3. 重排序 (Re-ranking)                           │
│    • bge-reranker-v2-m3模型精筛                 │
│    • 选出最相关的10个段落                        │
└─────────────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────────────┐
│ 4. 生成 (Generation)                             │
│    • 问题+10个段落→Qwen3-235B-A22B              │
│    • 基于参考资料生成答案                        │
│    • 标注引用来源                                │
└─────────────────────────────────────────────────┘
```

#### 第四阶段：AI智能体交互

## 四、核心创新点

### 1. 攻克"基因命名混乱"难题

**行业痛点**：
- 马铃薯参考基因组版本众多（DMv4.03, SolTub3.0等）
- 同一基因在不同版本有不同编号
- 容易出现"张冠李戴"的错误

**创新解决方案**：
- 利用**共线性分析**和**序列比对**
- 将所有旧版本基因ID统一映射到DMv8.1
- 建立基因ID与基因符号的对应关系库
- **标注历史错误符号**，确保用旧名字也能找到

**价值**：
- 科学家用任何旧名字都能找到准确位置
- 避免了研究方向性错误

### 2. 引入RAG架构消除"AI幻觉"

**通用AI的问题**：
- ❌ 没有专门学习马铃薯知识
- ❌ 容易产生"幻觉"，编造内容
- ❌ 没有引用来源，无法验证
- ❌ 科学家不敢信任

**本平台创新**：
- ✅ **RAG技术**（检索增强生成）
- ✅ 先检索真实文献，再生成答案
- ✅ 每条回答都标注引用来源
- ✅ 像"开卷考试"一样有据可查

### 3. 全流程数据"清洗与提纯"

**创新性技术路径**：

```
PDF文档 → MinerU → 纯文本
                      ↓
                 DeepSeek/Qwen
                      ↓
              结构化基因数据
                      ↓
                人工校验
                      ↓
              高质量数据库
```

**意义**：
- 展示了如何从陈旧PDF中挖掘结构化数据
- 为其他作物提供了可复制的技术范式

### 4. 全球首个马铃薯AI专业知识中心

- **不是简单数据库**，而是能理解、能回答、能推理的AI系统
- **专门针对马铃薯领域**训练
- **拒绝"幻觉"**，确保科学准确性
