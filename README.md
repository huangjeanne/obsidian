## 🔍 常见问题

### Q1: 为什么使用数字编码？
**A**: 数字编码有以下优势：
- 排序固定，目录始终按顺序显示
- 扩展性强，可以插入新的分类（15-16, 35-36...）
- 国际化，避免中英文混排问题

### Q2: 如何决定内容放在哪里？
**A**: 使用决策树（见"快速开始"部分），基本原则：
- 工作相关 → 20-29
- 生活相关 → 10-19
- 创作相关 → 60-69
- 学习相关 → 40-49
- 技术相关 → 30-39

### Q3: 文件命名有什么规范？
**A**: 遵循全局命名规范：
```
YYYYMMDD_主题_v版本.扩展名
```

示例：
- ✅ `20260228_水稻育种实验_v1.0.md`
- ✅ `20260228_AI提示词优化_v2.0.md`

### Q4: 如何迁移旧内容？
**A**:
1. 运行优化脚本：`bash 20260228_Obsidian目录优化_v1.0.sh`
2. 手动移动文件到对应分类
3. 更新文件中的双向链接
