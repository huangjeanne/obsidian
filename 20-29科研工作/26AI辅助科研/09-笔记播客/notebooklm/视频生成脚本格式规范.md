# 视频生成脚本格式规范

## 概述

本文档定义了自动化视频生成系统的Markdown脚本格式规范。该格式专为知识科普、教学视频和学术论文讲解设计，支持多角色配音、情感控制、动态背景生成和公式动画。

---

## 一、全局元数据块（YAML Frontmatter）

每个脚本文件必须以YAML格式的元数据块开始：

```yaml
---
title: "视频标题"
subtitle: "副标题（可选）"
author: "作者名称"
version: "1.0"

# 视频配置
duration: 180              # 预期时长（秒），1-5分钟 = 60-300秒
aspect_ratio: "16:9"       # 视频比例：16:9 或 9:16

# 风格配置
style: "academic"          # 风格预设：academic, tech, minimal, handdrawn, 3d
color_scheme: "blue"       # 配色方案：blue, green, red, warm, cool, custom

# 音频配置
voices:
  narrator:                # 旁白角色（可选）
    voice_id: "professional_male"
    emotion: "neutral"
    speed: 1.0

  teacher:                 # 教师角色
    voice_id: "friendly_female"
    emotion: "enthusiastic"
    speed: 1.0

  student:                 # 学生角色
    voice_id: "curious_male"
    emotion: "engaged"
    speed: 1.0

# 背景音乐（可选）
background_music:
  enabled: true
  style: "ambient"         # ambient, upbeat, dramatic, minimal
  volume: 0.3              # 音量 0.0-1.0

# 封面配置
cover:
  type: "generated"        # generated, custom
  style: "tech"            # 覆盖全局style
  title_display: true      # 是否显示标题
---
```

### 元数据字段说明

#### 必填字段
| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| `title` | string | 视频标题 | 任意文本 |
| `duration` | integer | 预期时长（秒） | 60-300 |
| `aspect_ratio` | string | 视频比例 | "16:9", "9:16" |
| `style` | string | 视觉风格 | "academic", "tech", "minimal", "handdrawn", "3d" |
| `voices` | object | 角色声音配置 | 至少1个角色 |

#### 可选字段
| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `subtitle` | string | 副标题 | 无 |
| `author` | string | 作者 | 无 |
| `version` | string | 版本号 | "1.0" |
| `color_scheme` | string | 配色方案 | "blue" |
| `background_music` | object | 背景音乐配置 | `{enabled: false}` |
| `cover` | object | 封面配置 | 见下方 |

#### 声音角色配置
```yaml
voices:
  role_name:              # 角色名称（自定义）
    voice_id: "string"    # Listenhub声音ID或预设名称
    emotion: "string"     # 默认情感
    speed: float          # 语速倍率，0.5-2.0，1.0为正常
```

**预设情感类型：**
- `neutral` - 中性
- `enthusiastic` - 热情
- `engaged` - 投入
- `curious` - 好奇
- `serious` - 严肃
- `friendly` - 友好
- `calm` - 平静
- `excited` - 兴奋

## 二、场景/分镜头格式

### 2.1 场景级别标记

使用二级标题（`##`）标记场景：

```markdown
## 场景1：开场引入

**元数据**：
- 角色：teacher
- 情感：friendly
- 时长：15秒
- 背景类型：dynamic

教师以友好的语气欢迎学生，介绍本节课的主题。

**视觉内容**：
```
类型：title_card
标题：相对论入门
副标题：改变时空的革命
动画：渐入，标题从中心放大
```

**动态背景**：
科技感粒子流动，深蓝色渐变背景
```
```

### 2.2 场景元数据字段

每个场景必须包含以下元数据块：

```markdown
**元数据**：
- 角色：teacher | student | narrator | teacher,student
- 情感：friendly | serious | curious | enthusiastic | neutral | calm
- 时长：<数字>秒
- 背景类型：static | dynamic | video | color
- 转场：none | fade | slide | zoom | wipe | dissolve
- 转场时长：0.5秒
```

#### 元数据详解

| 字段 | 说明 | 可选值 |
|------|------|--------|
| **角色** | 说话角色 | 单角色或多角色（用逗号分隔） |
| **情感** | 说话时的情感基调 | 见上方预设情感类型 |
| **时长** | 场景预估时长 | 5-60秒 |
| **背景类型** | 背景生成方式 | 见下方说明 |
| **转场** | 场景切换效果 | 见下方说明 |
| **转场时长** | 转场动画时长 | 0.3-2.0秒 |

#### 背景类型说明
| 类型 | 说明 | 生成方式 |
|------|------|----------|
| `static` | 静态背景 | Nano Banana Pro生成一次 |
| `dynamic` | 动态背景 | 每帧/每秒生成新背景 |
| `video` | 视频背景 | 使用预设视频素材 |
| `color` | 纯色背景 | CSS颜色值 |

#### 转场效果说明
| 效果 | 说明 | 适用场景 |
|------|------|----------|
| `none` | 无转场 | 直接切换 |
| `fade` | 淡入淡出 | 通用 |
| `slide` | 滑动 | 场景切换 |
| `zoom` | 缩放 | 强调重点 |
| `wipe` | 擦除 | 时间线推进 |
| `dissolve` | 溶解 | 柔和过渡 |

### 2.3 对话格式

```markdown
**对话**：

Teacher: 今天我们要学习一个有趣的概念，叫做"相对论"。

Student: 相对论？是不是爱因斯坦的那个理论？

Teacher: 没错！爱因斯坦的相对论彻底改变了我们对时间和空间的理解。
（停顿1秒）
让我们从一个简单的例子开始。
```

**对话规则：**
- 使用角色名称（来自元数据中的voices配置）
- 使用冒号（`:`）分隔角色和对话
- 空行表示自然停顿（约0.5-1秒）
- 使用`（停顿X秒）`明确标注停顿时长
- 使用`*动作*`标注肢体动作（可选）

## 三、视觉内容类型

### 3.1 视觉类型清单

| 类型标识 | 说明 | 动画复杂度 | Manim使用 |
|---------|------|-----------|-----------|
| `title_card` | 标题卡片 | 低 | 文字动画 |
| `text_bullet` | 文字列表 | 低 | 文字逐行显示 |
| `formula` | 数学公式 | 中 | 公式推导动画 |
| `chart` | 图表 | 中 | 图表绘制动画 |
| `diagram` | 示意图 | 中-高 | 自定义图形 |
| `scene` | 场景描述 | 高 | 场景构建 |
| `code` | 代码展示 | 中 | 代码高亮 |
| `quote` | 引用 | 低 | 引用样式 |
| `comparison` | 对比表格 | 中 | 表格动画 |
| `timeline` | 时间线 | 中 | 时间轴动画 |
| `process` | 流程图 | 中-高 | 流程绘制 |

### 3.2 视觉内容格式

```markdown
**视觉内容**：
```
类型：formula
标题：质能方程
内容：
- 初始状态：E = ?
- 推导步骤1：引入质量m
- 推导步骤2：引入光速c
- 最终结果：E = mc²
动画：逐字显示公式，高亮关键变量
布局：居中，大字体
颜色：公式为金色，背景深蓝
```
```

### 3.3 各类型的详细参数

#### Type: title_card（标题卡片）
```
类型：title_card
标题：主标题
副标题：副标题
动画：fade_in | zoom_in | slide_up | typewriter
布局：center | top | bottom
字体：大 | 中 | 小
```

#### Type: text_bullet（文字列表）
```
类型：text_bullet
标题：列表标题
项目：
- 项目1
- 项目2
- 项目3
动画：逐项显示 | 同时显示
布局：left | center | right
```

#### Type: formula（数学公式）
```
类型：formula
标题：公式名称
内容：
  LaTeX格式的完整公式
步骤：                     # 可选，用于推导过程
  - 步骤1描述
  - 步骤2描述
动画：步骤式 | 逐字 | 同时
布局：center | inline
强调：变量名或部分         # 高亮显示
```

#### Type: chart（图表）
```
类型：chart
图表类型：bar | line | pie | scatter | radar
标题：图表标题
数据：
  - 标签1: 数值1
  - 标签2: 数值2
  - 标签3: 数值3
动画：柱状生长 | 折线绘制 | 饼图展开
坐标轴：显示/隐藏
网格：显示/隐藏
```

#### Type: diagram（示意图）
```
类型：diagram
标题：图解标题
描述：图形详细描述
元素：
  - 元素1：位置和样式
  - 元素2：位置和样式
连接：
  - 元素1 -> 元素2：连接关系
动画：逐步构建 | 整体显示
```

#### Type: scene（场景描述）
```
类型：scene
描述：场景详细视觉描述
环境：室内/室外/抽象
人物：角色名称和位置
动作：主要动作描述
视角：正面/侧面/俯视/仰视
动画：入场 | 动作演示 | 出场
```

#### Type: code（代码展示）
```
类型：code
语言：python | javascript | etc.
内容：完整代码
动画：逐行高亮 | 语法高亮 | 打字机效果
布局：center | left
行号：显示/隐藏
```

#### Type: comparison（对比表格）
```
类型：comparison
标题：对比标题
维度：维度名称（如"优缺点对比"）
项目：
  - 项目1：
    - 属性1：值1
    - 属性2：值2
  - 项目2：
    - 属性1：值1
    - 属性2：值2
动画：逐行显示 | 逐列显示
```

## 四、动态背景规范

### 4.1 背景生成频率

```markdown
**动态背景**：
科技感粒子流动，深蓝色渐变背景
变化频率：每0.5秒
持续时间：场景全程
```

### 4.2 背景描述格式

**背景描述应包含：**

1. **基础风格**：颜色、纹理、氛围
2. **动态元素**：移动的粒子、光线、图案
3. **变化规则**：如何随时间变化
4. **与内容的关联**：背景如何配合视觉内容

**示例：**

```markdown
# 学术风格 - 静态背景
**动态背景**：
浅米色背景，细微纸张纹理
变化频率：静态（不变化）

# 科技风格 - 动态背景
**动态背景**：
深蓝色渐变，科技感几何图形浮动，光效扫过
变化频率：每秒
持续时间：场景全程
过渡方式：平滑渐变

# 手绘风格 - 动态背景
**动态背景**：
暖色调纸张纹理，手绘星星和线条缓慢移动
变化频率：每2秒
持续时间：场景全程
```

## 五、完整脚本示例

### 5.1 短示例（学术论文讲解）

```yaml
---
title: "RAG架构解析"
subtitle: "检索增强生成技术详解"
duration: 120
aspect_ratio: "16:9"
style: "tech"
color_scheme: "blue"

voices:
  narrator:
    voice_id: "professional_male"
    emotion: "neutral"
    speed: 1.0
---
```

```markdown
## 场景1：引入主题

**元数据**：
- 角色：narrator
- 情感：neutral
- 时长：20秒
- 背景类型：dynamic
- 转场：fade

**对话**：

Narrator: 检索增强生成，简称RAG，是大语言模型应用的一项关键技术。

**视觉内容**：
```
类型：title_card
标题：什么是RAG？
副标题：Retrieval-Augmented Generation
动画：typewriter（打字机效果）
布局：center
```

**动态背景**：
深蓝色科技感背景，数据流从四周向中心汇聚，形成RAG三个字母
变化频率：每秒

---

## 场景2：RAG架构解析

**元数据**：
- 角色：narrator
- 情感：neutral
- 时长：40秒
- 背景类型：static
- 转场：slide

**对话**：

Narrator: RAG系统包含三个核心组件：检索器、生成器和重排序器。

（停顿1秒）

首先，检索器将用户问题转换为向量，在向量数据库中查找相关文档。

**视觉内容**：
```
类型：diagram
标题：RAG工作流程
描述：从左到右的流程图
元素：
  - 用户问题：左侧圆形
  - 检索器：中间方形
  - 向量数据库：下方圆柱形
  - 重排序器：中间右侧方形
  - 生成器：右侧方形
连接：
  - 用户问题 -> 检索器：实线箭头
  - 检索器 -> 向量数据库：双向箭头
  - 检索器 -> 重排序器：实线箭头
  - 重排序器 -> 生成器：实线箭头
动画：逐步构建
布局：horizontal
```

**动态背景**：
深蓝色静态背景，科技感网格纹理

## 场景2：时间膨胀概念

**元数据**：
- 角色：teacher
- 情感：serious
- 时长：45秒
- 背景类型：static
- 转场：slide

**对话**：

Teacher: 让我们用数学来理解。这叫做"时间膨胀"。

（停顿2秒）

一个观察者看到的时间，与运动物体内部的时间，是不同的。

**视觉内容**：
```
类型：formula
标题：时间膨胀公式
内容：
  t' = t / √(1 - v²/c²)

步骤：
  - 步骤1：t'是运动物体中的时间
  - 步骤2：t是静止观察者的时间
  - 步骤3：v是物体速度
  - 步骤4：c是光速
  - 步骤5：当v接近c时，分母接近0
动画：步骤式，每步高亮对应部分
布局：center
强调：√(1 - v²/c²)
```

**动态背景**：
深蓝色静态背景，淡雅的时钟图案水印

## 九、最佳实践

1. **时长控制**：单个场景建议15-60秒，过长会失去观众注意力
2. **视觉多样性**：相邻场景使用不同的视觉类型
3. **情感节奏**：从引入到总结，情感应有起伏变化
4. **背景动态**：重要讲解场景使用静态背景，避免干扰
5. **公式动画**：复杂公式分步骤展示，每步清晰标注
6. **对话自然**：使用停顿和口语化表达，避免机械朗读感
7. **色彩对比**：确保文字与背景有足够对比度
8. **多语言考虑**：如需多语言，使用注释标注各语言版本
