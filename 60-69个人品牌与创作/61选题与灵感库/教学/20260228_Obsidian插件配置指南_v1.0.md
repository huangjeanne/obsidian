## 前置准备

### 1. 安装 Obsidian

**下载地址**: https://obsidian.md/download

**支持平台**:
- macOS
- Windows
- Linux
- iOS (iPad/iPhone)
- Android

### 2. 创建知识库（Vault）

1. 打开 Obsidian
2. 点击 "创建新知识库"
3. 选择知识库位置：
   - 推荐：`~/Documents/ObsidianVault` 或 `~/Library/Mobile Documents/iCloud~md~obsidian/Documents`
4. 命名知识库：如 "Jeanne-Second-Brain"

### 3. 启用第三方插件

1. 打开设置（⌘/Ctrl + ,）
2. 滚动到 "第三方插件"
3. 关闭 "安全模式"（允许安装社区插件）
4. 点击 "浏览" 搜索插件

## 必装插件配置

### 优先级1：基础框架插件（立即安装）

#### 插件1：Dataview ⭐⭐⭐⭐⭐

**功能**: 将笔记变为数据库，支持复杂查询

**安装步骤**:
1. 设置 → 第三方插件 → 浏览
2. 搜索 "Dataview"
3. 安装并启用

**配置项**:
```yaml
# Dataview 插件设置
Enable JavaScript Queries: true   # 启用 JS 查询
Render DataviewJS: true            # 渲染 DataviewJS
DataviewJS Timeout: 10000          # 超时时间
```

**快速测试**:
```markdown
```dataview
LIST
FROM ""
LIMIT 10
```
```

---

#### 插件2：Templater ⭐⭐⭐⭐⭐

**功能**: 强大的模板引擎，支持 JavaScript

**安装步骤**:
1. 设置 → 第三方插件 → 浏览
2. 搜索 "Templater"
3. 安装并启用

**配置项**:
```yaml
# Templater 插件设置
Template folder location: Templates
Date Format: YYYY-MM-DD HH:mm:ss
Time Format: HH:mm:ss

# 高级设置（重要）
Enable System Commands: true
Enable JavaScript System Commands: true
```

**创建模板文件夹**:
1. 在 Vault 根目录创建 `Templates` 文件夹
2. 在 Templates 中创建测试模板 `test.md`:
```markdown
---
title: <% tp.file.title %>
date: <% tp.date.now("YYYY-MM-DD") %>
---

# <% tp.file.title %>

创建时间: <% tp.date.now("YYYY-MM-DD HH:mm:ss") %>
```

**测试方法**:
1. 按 `⌘/Ctrl + P` 打开命令面板
2. 输入 "Templater: Insert Template"
3. 选择 `test.md`
4. 应该自动填充日期和时间

#### 插件3：Zotero Integration ⭐⭐⭐⭐⭐（科研必备）

**前置条件**:
1. 安装 [Zotero](https://www.zotero.org/download/)
2. 安装 Better BibTeX 插件（Zotero 插件）

**配置 Zotero**:
1. 打开 Zotero
2. 工具 → 插件 → 搜索 Better BibTeX
3. 安装后重启 Zotero
4. 编辑 → 首选项 → Better BibTeX
   - 引用键格式: `[auth][year]`
   - 字符集: Unicode

**配置 Obsidian 插件**:
1. 安装 Zotero Integration 插件
2. 配置项：
```yaml
# Zotero Integration 设置
Import Bibliography: true         # 导入参考文献
Import Abstracts: true            # 导入摘要
PDF Auto-import: true             # 自动导入 PDF 标注
Citation Style Template:          # 引用格式
```

**测试方法**:
1. 在 Zotero 中选择一篇文献
2. 按 `Cmd/Ctrl + Shift + D`（导出到 Obsidian）
3. 应该自动创建文献笔记

#### 插件7：Obsidian Git ⭐⭐⭐⭐

**功能**: 自动版本控制和备份

**前置条件**:
- 安装 Git（`brew install git` 或 `git-scm.org`）
- 创建 GitHub/GitLab 私有仓库

**配置 Git**:
```bash
# 1. 配置 Git 用户信息
git config --global user.name "Jeanne"
git config --global user.email "your-email@example.com"

# 2. 在 Obsidian Vault 中初始化 Git
cd ~/Documents/ObsidianVault
git init
git remote add origin https://github.com/username/vault-name.git
```

**配置 Obsidian Git 插件**:
```yaml
# Obsidian Git 插件设置
Auto Commit Interval: 30          # 自动提交间隔（分钟）
Auto Commit Message:
  Auto backup: {{date}}
Auto Push Interval: 60            # 自动推送间隔（分钟）
Pull On Startup: true             # 启动时拉取
Pull Before Push: true            # 推送前拉取
```

**测试方法**:
1. 修改任意笔记
2. 等待 30 分钟或手动触发（⌘/Ctrl + P → "Obsidian Git: Commit")
3. 检查 Git 仓库是否有新提交

## 核心插件详解

### Dataview 深度使用

#### 查询语法

**基础查询**:
```dataview
LIST
FROM "03-Resources/Literature"
WHERE contains(tags, "literature")
SORT year DESC
```

**表格查询**:
```dataview
TABLE file.link as "文献", authors as "作者", year as "年份", status as "状态"
FROM "03-Resources/Literature"
WHERE contains(tags, "literature")
SORT year DESC
```

**任务查询**:
```dataview
TASK
FROM #task
WHERE due >= date(today) AND due <= date(today) + dur(7 days)
WHERE !completed
GROUP BY due
```

**数据统计**:
```dataview
TABLE length(rows) as "数量"
FROM "03-Resources/Literature"
GROUP BY tags
```

#### 常用查询示例

**查看所有项目**:
```dataview
TABLE file.mtime AS "最后更新", status AS "状态", priority AS "优先级"
FROM #project
WHERE status != "completed"
SORT priority ASC
```

**查看本周阅读的文献**:
```dataview
TABLE file.link as "文献", authors as "作者"
FROM "03-Resources/Literature"
WHERE date(created) >= date(today) - dur(7 days)
SORT created DESC
```

**查看所有未完成任务**:
```dataview
TASK
FROM #task
WHERE !completed
GROUP BY due
```

### Templater 深度使用

#### 动态变量

**日期时间**:
```markdown
<% tp.date.now("YYYY-MM-DD") %>
<% tp.date.now("YYYY-MM-DD HH:mm:ss") %>
<% tp.date.now("dddd") %>  <!-- 星期几 -->
```

**文件信息**:
```markdown
<% tp.file.title %>         <!-- 文件名 -->
<% tp.file.path %>           <!-- 文件路径 -->
<% tp.file.folder %>         <!-- 所属文件夹 -->
```

**用户输入**:
```markdown
<% tp.system.prompt("请输入文献标题") %>
<% tp.system.suggester(["水稻", "玉米", "小麦"], "选择作物类型") %>
```

#### 动态模板示例

**根据文件夹自动设置标签**:
```markdown
<%*
// 获取当前文件夹
const folder = tp.file.folder();

// 根据文件夹设置标签
if (folder.includes("水稻")) {
  tR += "\ntags: [rice, breeding]";
} else if (folder.includes("玉米")) {
  tR += "\ntags: [corn, breeding]";
} else if (folder.includes("小麦")) {
  tR += "\ntags: [wheat, breeding]";
} else {
  tR += "\ntags: [general]";
}
%>
```

**自动生成唯一 ID**:
```markdown
<%*
// 生成唯一 ID
const id = tp.date.now("YYYYMMDDHHmmss");
tR += "\nid: " + id;
%>
```

## 测试验证

### 集成测试脚本

**完整测试流程**:

```python
#!/usr/bin/env python3
"""
Obsidian 知识管理体系集成测试
"""

import requests
from datetime import datetime

class ObsidianTest:
    def __init__(self, api_key, base_url="http://localhost:27124"):
        self.api_key = api_key
        self.base_url = base_url
        self.headers = {
            "Authorization": f"Bearer {api_key}",
            "Content-Type": "application/json"
        }

    def test_connection(self):
        """测试 API 连接"""
        try:
            response = requests.get(f"{self.base_url}/", headers=self.headers)
            if response.status_code == 200:
                print("✅ API 连接成功")
                return True
            else:
                print(f"❌ API 连接失败: {response.status_code}")
                return False
        except Exception as e:
            print(f"❌ 连接错误: {e}")
            return False

    def test_create_note(self):
        """测试创建笔记"""
        test_content = f"""---
title: 测试笔记
created: {datetime.now().strftime("%Y-%m-%d")}
tags: [test]
---

# 测试笔记

这是一个测试笔记，用于验证 Obsidian API 集成。

## 测试项目
- [x] API 连接
- [ ] 创建笔记
- [ ] 搜索笔记
- [ ] 删除笔记
"""

        try:
            response = requests.put(
                f"{self.base_url}/notes/06-Inbox/test-note.md",
                headers=self.headers,
                json={"content": test_content}
            )

            if response.status_code == 200:
                print("✅ 创建笔记成功")
                return True
            else:
                print(f"❌ 创建笔记失败: {response.status_code}")
                return False
        except Exception as e:
            print(f"❌ 创建笔记错误: {e}")
            return False

    def test_search_notes(self):
        """测试搜索笔记"""
        try:
            response = requests.get(
                f"{self.base_url}/search",
                headers=self.headers,
                params={"query": "#test"}
            )

            if response.status_code == 200:
                results = response.json().get("results", [])
                if len(results) > 0:
                    print(f"✅ 搜索成功，找到 {len(results)} 条结果")
                    return True
                else:
                    print("⚠️  搜索成功，但没有找到结果")
                    return True
            else:
                print(f"❌ 搜索失败: {response.status_code}")
                return False
        except Exception as e:
            print(f"❌ 搜索错误: {e}")
            return False

    def test_dataview_query(self):
        """测试 Dataview 查询"""
        # 这个需要手动在 Obsidian 中验证
        print("📝 请在 Obsidian 中运行以下 Dataview 查询验证：")
        print("```dataview")
        print('LIST FROM #test')
        print("```")
        return True

    def run_all_tests(self):
        """运行所有测试"""
        print("🧪 开始 Obsidian 集成测试...\n")

        tests = [
            ("API 连接", self.test_connection),
            ("创建笔记", self.test_create_note),
            ("搜索笔记", self.test_search_notes),
            ("Dataview 查询", self.test_dataview_query),
        ]

        results = []
        for test_name, test_func in tests:
            print(f"\n📋 测试: {test_name}")
            result = test_func()
            results.append((test_name, result))

        # 汇总结果
        print("\n" + "=" * 50)
        print("📊 测试结果汇总:")
        passed = sum(1 for _, r in results if r)
        total = len(results)
        print(f"通过: {passed}/{total}")

        for test_name, result in results:
            status = "✅ 通过" if result else "❌ 失败"
            print(f"  {test_name}: {status}")

        return passed == total

if __name__ == "__main__":
    # 替换为你的 API Key
    api_key = "your-api-key-here"

    tester = ObsidianTest(api_key)
    success = tester.run_all_tests()

    if success:
        print("\n🎉 所有测试通过！")
    else:
        print("\n⚠️  部分测试失败，请检查配置")
```

**运行测试**:
```bash

```

## 常见问题

### Q1: 插件安装失败

**问题**: 无法安装社区插件

**解决方案**:
1. 检查网络连接
2. 尝试使用 VPN（某些地区可能被墙）
3. 手动下载插件：
   - 访问 [GitHub Releases](https://github.com/search?q=obsidian+plugin)
   - 下载 `.zip` 文件
   - 解压到 Vault 的 `.obsidian/plugins/` 文件夹

### Q2: Dataview 查询无结果

**问题**: Dataview 查询返回空结果

**解决方案**:
1. 检查笔记是否有 YAML frontmatter
2. 检查标签格式（应该是 `tags: [tag1, tag2]` 或 `tags: tag1`）
3. 检查查询语法是否正确
4. 尝试简化查询（如 `LIST FROM ""`）

### Q3: Zotero Integration 无法连接

**问题**: Obsidian 无法连接到 Zotero

**解决方案**:
1. 确保 Zotero 正在运行
2. 安装 [Zotero HTTP Server](https://github.com/doczoda/zotero-server)
3. 配置 CORS（如果使用 REST API）
4. 重启 Obsidian 和 Zotero

### Q4: Obsidian Git 推送失败

**问题**: Git 自动推送失败

**解决方案**:
1. 检查 Git 配置：`git config --list`
2. 检查远程仓库：`git remote -v`
3. 测试手动推送：`git push origin main`
4. 检查 SSH 密钥或 GitHub Token
5. 查看插件日志：Obsidian 设置 → Obsidian Git → Open Log

### Q5: Local REST API 无法连接

**问题**: Python 脚本无法连接到 Obsidian API

**解决方案**:
1. 确认 Obsidian 正在运行
2. 检查端口号（默认 27124）
3. 检查 API Key 是否正确
4. 测试连接：`curl -H "Authorization: Bearer your-key" http://localhost:27124/`
5. 检查防火墙设置
