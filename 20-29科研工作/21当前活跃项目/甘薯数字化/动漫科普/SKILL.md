# 农业数字孪生漫画脚本生成系统 (中文注解版 - v8.0)

> **版本**: v8.0 (Cinematic Digital Twin Edition)
> **代号**: 电影级数字孪生
> **核心理念**: 从"绘制连环画"到"构建数字孪生世界"

**三大铁律（THREE IRON RULES）**:
1. **沉浸优先 (Immersion First)** ⚠️ MANDATORY: 用镜头讲故事，而非用对话框讲课
2. **物理真实 (Physically Correct)** ⚠️ MANDATORY: 光影、透视、材质必须符合物理定律，数据层（UI）与物理层（世界）严格分离
3. **冲突开场 (Cold Open)** ⚠️ MANDATORY: 拒绝平铺直叙，用视觉冲突瞬间抓住注意力

**v8.0 核心特性** (从v7.2升级):
- **数字孪生叙事系统** ⭐: 构建基于真实物理法则的数字孪生世界，使用电影镜头语言记录农业生产
- **电影级渲染协议** ⭐: UE5渲染风格、8K分辨率、照片写实、景深效果、体积光照
- **冷开场模式** ⭐: Panel 1强制展示冲突/困境/挑战，禁止"大家好"式开场
- **动态分镜类型** ⭐: 三种分镜模式（标准叙事/POV沉浸/延时蒙太奇）
- **空间锚点系统** (v7.3): 防止自由机位穿梭导致的空间迷失
- **介质化UI系统** (v7.3): UI依附于物理实体（平板屏幕），通过过肩镜头呈现
- **HUD悬浮文字** ⭐: 替代传统对话气泡，高科技感悬浮字幕，半透明引线
- **光场锁定系统** (v7.3): 统一左上方10-11点钟主光源，屏幕补光效果
- **幻影透视系统** (v7.3): 实景向工程蓝图平滑过渡，替代传统剖面图
- **角色渲染开关** (v7.3): POV视角自动切换手部模型
- **胶卷结构系统** (v7.3): 延时蒙太奇的4分屏静态机位生成
- **声音设计系统** ⭐: 极简对话、微小说体拟声词、环境音纹理
- **深度研究系统** (v5.1保留): 参考图深度解读 + 网络文献搜索
- **数字资产定义** ⭐: 邱教授（无设备/传统经验）+ 林场长（银色平板/AR单镜片）

**v8.0 与 v7.2 的核心差异**:
- ❌ **移除**: 对话气泡系统、卡通/插画风格、平铺直叙开场
- ✅ **新增**: 电影感渲染、HUD悬浮文字、冷开场冲突、声音设计
- ⚠️ **重大跃迁**: 从"科普漫画"到"数字孪生世界"

**目的**: 为数字孪生叙事建立空间参照系和物理资产基础。

##### A. 空间锚点定义 (Spatial Anchors)

**设计目的**: 防止自由机位穿梭时导致的空间迷失，确保无论镜头如何运动，场景的地理逻辑始终一致。

**必须定义的全局地标**（在脚本开头强制声明）:

```markdown
**【GLOBAL SPATIAL ANCHORS - 空间锚点系统】** ⭐v8.0 MANDATORY:

**Anchor A - 北/远景地标**: "红砖房" (Red Brick House)
- 永久位置: 田地北端/远景背景
- 视觉特征: 红砖墙、灰瓦屋顶、两层建筑
- 参考作用: 指示北方/背景深度
- 镜头参照: 所有镜头需描述相对位置（如"背对红砖房"、"红砖房在画面右侧"）

**Anchor B - 功能/近景地标**: "灌溉主渠" (Irrigation Main Canal)
- 永久位置: 与田垄平行，垄沟东侧
- 视觉特征: 混凝土沟渠，有水流/干涸状态
- 参考作用: 指示田垄走向/水平基准
- 镜头参照: 确保镜头旋转时渠的位置正确变化

**空间布局**:
- 垄向: 南北走向（从近处向远处延伸）
- 红砖房: 北端背景（在所有分镜的远景中可见）
- 灌溉渠: 垄沟东侧（平行于田垄）
```

**每个分镜必须包含的锚点描述**:

```markdown
**Camera Orientation Relative to Anchors**:
- 镜头朝向: 面向北方（背景远端可见红砖房）
- 相对位置: 站在垄沟中，顺着垄沟看向北方
- 锚点可见性: 红砖房在背景中模糊可见
```

**设计目的**: 为"介质化UI"（Diegetic UI）建立物理基础，UI必须依附于真实存在的设备。

**角色数字资产配置**:

```markdown
**【DIGITAL ASSETS - 数字资产定义】** ⭐v8.0 MANDATORY:

**邱教授 (Professor Qiu)**:
- **电子设备**: 无任何电子设备
- **角色定位**: 传统经验的化身
- **呈现方式**: 仅用眼睛观察、手势比划、口头讲解

**林场长 (Farmer Lin)**:
- **电子设备**: 银色加固型平板电脑 (Rugged Silver Tablet)
- **可选设备**: AR单镜片（戴在左眼，视科技感需求定）
- **设备功能**: 显示数据图表、技术步骤、土壤墒情监测
- **呈现方式**: 通过过肩镜头（OTS）展示屏幕内容
- **屏幕内容**: 高对比度数据图表、矢量插画、蓝白配色

**田间传感器 (Field Sensors)**:
- **类型**: 橙黑配色土壤墒情监测仪
- **位置**: 插在土壤中，分布在田垄上
- **功能**: 实时监测土壤湿度、温度、养分

```

**介质化UI渲染规则**:

```markdown
**Diegetic UI Rendering Rules**:
- ❌ 禁止: 空中悬浮的抽象图表

- ✅ 细节: 屏幕自发光微弱照亮角色脸部
- ✅ 反光: 屏幕蓝光在角色眼镜/脸上的反射
```

#### 1.1 叙事结构：冷开场模式 (⚠️ v8.0 新增 - 必须执行)

**设计理念**:
拒绝平铺直叙，用视觉冲突瞬间抓住注意力。用镜头讲故事，而非用对话框讲课。

**Panel 1 强制规则** (必须遵守):

```markdown
**【COLD OPEN - 冷开场模式】** ⭐v8.0 MANDATORY:

**状态**: 静默 (Silence)
**内容**: 展示冲突 (Conflict)、困境 (Problem) 或高难度挑战
**禁止**:
- ❌ 禁止出现"大家好"、"今天我们讲什么"等开场白
- ❌ 禁止人物直接对话介绍主题
- ❌ 禁止平淡的环境展示

**示例**:
✅ 正确示范:
- 特写镜头展示一株叶片发黄、根部腐烂的红薯苗
- 背景是林场长焦急模糊的身影
- HUD悬浮文字：*"林场长，这片甘薯怎么了？"*

✅ 正确示范:
- 俯视镜头展示大面积干旱龟裂的土壤
- 几株萎蔫的薯苗孤零零地立着
- HUD悬浮文字：*"旱情持续，如何保苗？"*

❌ 错误示范:
- 邱教授微笑着说："大家好，今天我们来学习甘薯栽插技术。"
- 平淡展示一片健康的甘薯田
```

**质量检查清单**:
- [ ] Panel 1是否展示了冲突/困境/挑战？
- [ ] Panel 1是否避免了平铺直叙的开场白？
- [ ] Panel 1是否使用视觉冲突抓住注意力？
- [ ] Panel 1是否保持静默（无大量对话）？

#### 1.2 动态分镜类型 (⚠️ v8.0 新增 - 必须混合使用)

**设计理念**:
根据叙事需求选择最佳的分镜类型，确保视觉多样性和叙事节奏。

**三种分镜类型** (必须在脚本中混合使用):

##### A. 标准叙事 (Standard Cinematic)

```markdown
**【STANDARD CINEMATIC - 标准叙事】**:

**视角**: 第三人称，电影感构图
**常用技巧**:
- 过肩镜头 (Over-The-Shoulder Shot, OTS)
- 低角度仰拍 (Low-Angle Shot)
- 荷兰角 (Dutch Angle, 增加张力)
- 三分法则构图 (Rule of Thirds)

**用途**:
- 交代人物关系
- 展示整体环境
- 技术讲解场景

**提示词示例**:
"Cinematic wide shot, medium lens, natural lighting from top-left."
```

##### B. 第一人称沉浸 (POV Mode)

```markdown
**【POV MODE - 第一人称沉浸】**:

**触发条件**: 展示具体手部操作（栽插、切割、触摸）
**渲染规则**:
- ✅ 角色: 隐去全身
- ✅ 手部: "First-person view. Visible hands only. Male hands, slightly tanned, black wristwatch on left wrist."
- ✅ 动作: 极度清晰的手部与物体交互细节

**提示词示例**:
"POV shot. First-person view. Male hands slightly tanned, black wristwatch on left wrist. Hands holding sweet potato seedling, inserting into soil at 45-degree angle. Extreme detail on finger movements and soil texture."
```

##### C. 延时蒙太奇 (Time-lapse Montage)

```markdown
**【TIME-LAPSE MONTAGE - 延时蒙太奇】**:

**触发条件**: 展示生长周期、时间流逝
**结构**: 单一宽幅分镜，内部切分为4个垂直等分画面 (Vertical Slices)
**约束**: 摄像机位置绝对锁定 (Static Camera)，背景不变，只有主体作物发生变化

**4个切片内容**:
- Slice 1 (左): 栽插当天 - 幼苗直立，土壤湿润
- Slice 2 (中左): 20天后 - 藤蔓延伸，覆盖30%
- Slice 3 (中右): 60天后 - 完全封垄，土壤不可见
- Slice 4 (右): 采收期 - 剖面图，薯块肥大

**提示词示例**:
"Filmstrip montage - 4 vertical slices, static camera. Slice 1: Day 0 seedling planting. Slice 2: Day 20 vines extending. Slice 3: Day 60 full canopy closure. Slice 4: Harvest with cross-section showing tubers. Uniform lighting across all slices."
```

**质量检查清单**:
- [ ] 脚本中是否混合使用了3种分镜类型？
- [ ] POV分镜是否正确隐藏了全身，仅展示手部？
- [ ] 延时蒙太奇是否使用了静态机位，仅主体变化？
- [ ] 标准叙事是否使用了电影感构图？

#### 1.3 声音设计 (⚠️ v8.0 新增 - 极简原则)

**设计理念**:
对话极简，用环境音纹理增强沉浸感。

**三大声音规则**:

##### A. 对话极简 (Minimalist Dialogue)

### 7. 幻影透视（Phantom View）系统 ⚠️ v7.3新增 - 渐变过渡

**设计理念**：
- **不是"硬切换"**：实景与线条图之间有平滑的过渡带
- **不是"统一风格"**：地上部分是照片写实，地下部分是技术线条图
- **渐变融合**：从写实（Photorealistic）→ 渐变（Gradient Fade）→ 线条图（Wireframe）

**应用场景**：
- 剖面俯视图（Cross-section Top-down）
- 地上+地下的双层叙事

**Phantom View 渲染模板**（必须在剖面图中使用）：

```markdown
**Cross-section View - 幻影透视 ⭐v7.3 Phantom View**:

**A. 地上部分（Above Ground） - 写实渲染**:
- **渲染风格**: Photorealistic rendering（照片写实）
- **光影**: 自然阳光（Natural sunlight from upper left）
- **纹理**: 详细的肌理（Detailed textures - 土壤颗粒、植物叶脉、布料质感）
- **色彩**: 全彩（Full color with rich shading）
- **体积感**: 丰富的光影层次（Rich light and shadow）

**B. 过渡带（Transition Zone）- 渐变消融**:
- **位置**: 地表线（Soil surface line）
- **宽度**: 约2-3厘米的过渡带
- **效果**:
  - 从写实逐渐淡化为线条图（Gradient fade to schematic）
  - 颜色饱和度降低（Desaturated colors）
  - 细节逐渐简化（Details gradually simplify）
  - 轮廓线逐渐强化（Outlines gradually strengthen）
- **实现**: Alpha渐变或层叠蒙版（Alpha gradient or layered mask）

**C. 地下部分（Below Ground）- 技术线条图**:
- **渲染风格**: 技术建筑蓝图风格（Technical architectural blueprint style）
- **线条**: 白色线条框架（White wireframe lines）
- **填充**: 深色土壤背景（Dark soil background - RGB 80, 50, 30）
- **光影**: 最小化（Minimal shading - 仅必要的阴影突出深度）
- **清晰度**: 矢量美学（Clean vector aesthetics）
- **标注**: 工程制图风格（Engineering drawing style）

**关键要素**:
1. **过渡带**：不是"一刀切"，而是"渐变融合"
2. **统一透视**：地下线条图的透视灭点（Vanishing Point）必须与地上的田垄透视灭点重合
3. **风格对比**：地上写实 vs 地下抽象，形成视觉冲击
4. **自然过渡**：读者能够理解"这是同一个东西的不同层面"
```

**过渡带描述模板**：
```markdown
**Transition Zone（过渡带）**:
- 从地表线向下2-3cm:
  - 颜色饱和度: 100% → 0%（渐变）
  - 细节密度: 高 → 低（渐变）
  - 轮廓线: 无 → 明显（渐变）
  - 阴影: 丰富 → 最小化（渐变）
- 效果: 自然过渡，无生硬边界
```

**输出格式**：
```markdown

**【Above Ground - Photorealistic】**:
- 邱教授弯腰，右手进行栽插操作
- 自然阳光从左上方照射，产生丰富的阴影
- 土壤颗粒感清晰，植物叶脉可见
- 衣物褶皱自然，面部结构清晰

**【Transition Zone - Gradient Fade】**:
- 地表线向下2-3cm的过渡带
- 从写实逐渐淡化为技术线条图
- 颜色饱和度降低，细节逐渐简化
- 轮廓线逐渐强化，无生硬边界

**【Below Ground - Blueprint Style】**:
- 白色线条框架（Wireframe lines）
- 深色土壤背景（Dark background RGB 80, 50, 30）

- 最小化阴影（Minimal shading）
```

### 8. 光场锁定（Light Field Locking）系统 ⚠️ v7.3新增 - 光影统一

**设计理念**：
- **拼贴感来源于光影不统一**
- **所有分镜（包括画中画）的光照必须一致**
- **透视灭点必须统一**

**三大检查项**（必须执行）：

#### A. 主光源方向检查 ⚠️ 强制

**检查标准**：
- 所有分镜的主光源方向：统一指向**左上方（10-11点钟方向）**
- 包括：人物的鼻影、服装的褶皱阴影、物体的投影
- 包括：画中画的边框高光、屏幕反光

**检查方法**：
```markdown
- [ ] 人物鼻影：指向右下方（光源在左上）
- [ ] 衣物阴影：一致的光影方向
- [ ] 地面投影：一致的光影方向
- [ ] 作物阴影：一致的光影方向
- [ ] 画中画边框：高光在左上，阴影在右下
- [ ] 屏幕反光：反光方向与主光源一致
```

#### B. 屏幕自发光检查 ⚠️ 细节真实感

**应用场景**：当有平板/手机特写时

**检查标准**：
- 屏幕自发光（Screen glow）微弱照亮角色的脸部或手指
- 屏幕光色温与屏幕内容一致（如：蓝色屏幕 → 蓝色补光）
- 屏幕反光（Screen reflection）在角色的眼镜、脸部、衣服上

**描述模板**：
```markdown
**Screen Glow Effect（屏幕发光效果）**:
- 屏幕蓝光微弱照亮林场长的脸和手指
- 蓝色反射光在林场长的脸颊和眼镜上
- 增加真实感和细节丰富度
```

#### C. 透视统一检查 ⚠️ 视觉一致性

**检查标准**：
- 地上景物（田垄、植株）的透视灭点（Vanishing Point）
- 地下线条图（根系、土壤分层）的透视灭点
- 两者必须重合

**描述模板**：
```markdown
**Perspective Consistency（透视统一）**:
- 地上田垄的透视灭点：画面右上方向
- 地下线条图的透视灭点：与地上田垄一致
- 统一的透视增强视觉连贯性
- 避免"拼贴感"
```

**质量检查清单**（必须执行）：
```markdown
**光场锁定检查清单**:

**Panel [编号]**:
- [ ] 主光源方向：左上方（10-11点钟）
- [ ] 人物鼻影：指向右下方
- [ ] 物体投影：方向一致
- [ ] （如有屏幕）屏幕自发光：照亮角色脸部/手指
- [ ] （如有屏幕）屏幕反光：与主光源方向一致
- [ ] （如有剖面）透视灭点：地上地下统一
```

## 重要限制说明

### Gemini API 限制

1. **固定输出尺寸**: 1024x1024，无法自定义画幅
2. **不支持 --cref 参数**: 需要使用详细的角色描述确保一致性
3. **中文文本长度**: 单个气泡建议不超过20字，避免拥挤

### 最佳实践

1. **角色一致性**: 每个分镜重复详细描述角色特征
2. **气泡位置**: 明确指定气泡位置（top/left/right），防止遮挡人脸
3. **对话长度**: 控制中文对话长度，确保清晰可读
4. **统一画幅**: 所有分镜使用 4:3 比例
